# Development Rules

- If a request requires broad or many-file changes, ask the user before proceeding.
- Do not put all code in a single file. Split code by feature or responsibility into modules.
- When a request is unclear, do not infer and execute immediately. First restate your understanding and ask for confirmation or clarification.
- Before committing or pushing Markdown documents to GitHub, translate Korean documents into English while preserving their meaning and project context.
- Do not commit Korean public-facing documentation unless the user explicitly asks to preserve the Korean original.
