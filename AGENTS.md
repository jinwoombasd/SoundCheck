# Agent Rules

- Before committing or pushing documents to GitHub, make sure public-facing Markdown documents are written in English.
- If a Markdown document is currently in Korean, translate it into English before it is committed or pushed.
- Preserve the meaning, decisions, dates, technical terms, and project context during translation.
- Do not upload secrets, tokens, local state files, logs, or temporary runtime files.
- If Korean source text must be preserved for a specific reason, ask the user before committing it.
