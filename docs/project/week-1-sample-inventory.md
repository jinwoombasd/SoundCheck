## Purpose

Track the first 10 church broadcast speech samples before the Week 2 file analyzer is implemented.

This inventory is documentation-only. It does not require copyrighted audio, live Slack process changes, token updates, or runtime audio processing.

## Collection Rules

- Use speech-only material from sermons, hosts, or pre-service microphone checks.
- Do not use copyrighted commercial audio for external demos.
- Keep raw files outside the public repository unless the usage rights are confirmed.
- Record enough context for later analysis without storing private church or attendee information.
- Label each bad sample with one primary problem from `week-1-sound-criteria.md`.

## Sample Storage Guidance

Use a local or private storage location for raw audio until sharing rights are confirmed.

Suggested private layout:

```text
samples/
  week-1/
    good/
    bad/
```

If sample files are ever added to the repository, confirm first that they are allowed to be shared.

## Minimum Metadata

Each sample should include:

- Sample ID
- Good or bad status
- Source type
- One primary problem for each bad sample
- A short reviewer note explaining the label
- Whether sharing rights are confirmed

## Review Checklist

- The five good samples are understandable and do not have obvious level, clipping, tonal-balance, noise, or stereo issues.
- The five bad samples cover several different first-pass problem types.
- No private attendee information is included in public notes.
- No raw audio is committed unless usage rights are confirmed.
- The report template can be filled out for at least one good sample and one bad sample.
